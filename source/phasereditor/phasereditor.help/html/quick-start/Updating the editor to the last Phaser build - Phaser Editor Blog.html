Updating the editor to the last Phaser build

We are editing this document right now

Each time we release Phaser Editor will support the Phaser version of at the meoment. However, in the last time Phaser is updated in small periods of time, what makes it really hard to keep the editor linked to the last Phaser build.

For that reason we created what we call the Phaser Bunde. It is a repository with the data used by many components of the editor that requires the Phaser source, examples and metadata.

The Phaser data is used in the editor when you create a project (it adds the phaser.js) or when you select a Phaser Example template (it fetch the source of the Phaser example) or when you open Chains, it uses metadata generated from the Phaser sources, or in the JavaScript editor to auto-complete the Phaser API, etc...

So if the user wants to enjoy the last Phaser version in all the features of the editor, what he has to do is to download or clone (or update) the Phaser Bundle repository, and point it from the editor.

Link the editor with a Phaser Bundle

When a new Phaser version is released, we update the Phaser Bundle and make it available for the public (it can take a week).

These are the steps you have to follow to include the bundle in the editor:

  1. Download, clone or update the Phaser Bundle repo.
  2. Open the Phaser preferences page: Window > Preferences > Phaser Editor > Phaser.
  3. Check the Use the Built-in Phaser bundle or set the External Phaser Bundle path.
  4. It will ask for an application restart.
